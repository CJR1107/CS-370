## CS 370

## Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?

For Project Two, I worked on a pirate intelligent agent for a treasure hunt game. The goal was to train the pirate to find the treasure using reinforcement learning and a neural network. The starter code included the maze environment, the experience storage, and parts of the Jupyter Notebook setup. That provided code helped define how the maze worked, how the agent interacted with it, and how past experiences were stored during training.

The code I created focused on completing the deep Q-learning process in the notebook. This included the training loop, action selection, exploration and exploitation, storing experiences, training the model, and checking whether the agent improved over time. The agent was not given the correct path directly. Instead, it learned from rewards, penalties, and repeated attempts.

## What do computer scientists do and why does it matter?

Computer scientists solve problems by designing, building, testing, and improving technology-based systems. Their work matters because software affects almost every part of modern life, including business, healthcare, education, transportation, security, and entertainment. Computer scientists do more than write code; they also think about how systems should work, how users will interact with them, and how to make them reliable and secure.

This project showed how computer science can solve problems where the answer is not obvious at first. The pirate agent had to learn better decisions over time, which connects to real-world uses such as navigation, robotics, automation, and intelligent decision-making systems.

## How do I approach a problem as a computer scientist?

I approach a problem by breaking it into smaller parts. First, I look at the goal, the inputs, the expected outputs, and the rules the system has to follow. In this project, the main goal was getting the pirate to the treasure, but that included smaller steps like understanding the maze, choosing actions, storing previous experiences, and updating the model through training.

I also learned that testing and adjusting are important parts of the process. The first solution is not always the best solution, especially in artificial intelligence. A model may need more training, better parameters, or a better balance between exploration and exploitation.

## What are my ethical responsibilities to the end user and the organization?

My ethical responsibilities are to create software that is reliable, secure, and fair. For the end user, this means the system should work as expected, protect user data, and avoid causing harm. Even though this project was a game, the same ideas apply to real AI systems because intelligent agents can make unexpected decisions if they are not trained or tested properly.

For the organization, my responsibility is to write code that is maintainable, tested, and aligned with the project’s purpose. I should focus not only on making the program run, but also on making sure it is understandable, dependable, and responsible. In AI projects, developers also have to consider accuracy, bias, security, and accountability.
